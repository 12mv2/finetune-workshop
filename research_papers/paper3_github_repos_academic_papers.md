# GitHub Repositories with Links to Academic Papers: Public Access, Traceability, and Evolution

## Overview
This large-scale empirical study examines 20,278 links to academic papers in GitHub repositories, analyzing public access availability, traceability between papers and repositories, and how these relationships evolve over time.

## Research Scope
The study uses a mixed-methods approach to investigate:
1. **Public Access** - Availability of referenced papers outside paywalls
2. **Traceability** - Bidirectional linking between repositories and academic papers
3. **Evolution** - How repository-paper relationships change over time

## Key Findings

### Public Access to Academic Papers
- **Majority Are Open Access** - A vast majority of referenced academic papers in GitHub repositories are publicly accessible
- **Academic Community Affiliation** - These repositories tend to be affiliated with academic research communities
- **Against the Trend** - Despite referencing papers not being typical practice in GitHub repositories, academic paper references are common in research-oriented repos

### Repository Characteristics
- **Machine Learning Dominance** - References to papers are frequently made in machine learning-related GitHub repositories
- **Author Participation** - 40% of referenced academic papers in GitHub repositories are referenced by the paper authors themselves
- **Multi-Language Impact** - The most referenced papers are cited by repositories written in different programming languages

### Traceability Issues
- **One-Way References** - More than half of the referenced papers do not link back to any repository
- **Top-Tier Venues** - Academic papers from top-tier Software Engineering venues are not likely to reference a repository
- **When They Do Link** - When top-tier papers do reference code, they usually link to a GitHub software repository

### Citation Impact
- **Highly-Cited Papers** - The most referenced papers in GitHub repositories are also highly-cited in academic literature
- **Cross-Domain Influence** - These papers influence development across multiple programming languages and domains

## Implications for Research Reproducibility

### Positive Aspects:
1. **Increased Transparency** - Linking papers to code improves research transparency
2. **Reproducibility** - Availability of implementation code aids in reproducing research results
3. **Open Science** - Trend supports open science principles

### Challenges:
1. **Incomplete Traceability** - Many papers lack bidirectional links to their code
2. **Academic Resistance** - Top-tier venues don't consistently require code availability
3. **Maintenance** - Repository evolution may break links or reduce code usability over time

## Recommendations

### For Researchers:
- Link academic papers to their GitHub repositories
- Maintain repositories after publication
- Document code clearly for reproducibility

### For Publishers:
- Encourage or require code availability for published papers
- Implement policies for linking papers to repositories
- Support reproducibility through code review

### For the Community:
- Standardize practices for linking papers and code
- Develop tools to improve traceability
- Promote open science culture

## Methodology Note
The study employed large-scale data mining of GitHub repositories, analyzing:
- Repository metadata
- Academic paper references
- Citation patterns
- Programming language distribution
- Temporal evolution patterns
